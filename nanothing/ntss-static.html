Static Temperature Monitoring

The world’s first standard for autonomous static temperature monitoring

NTSS for static temperature monitoring is a much simpler version of the NTSS standard, designed for use cases that don’t involve transportation


See what you can do with NTSS for Static Monitoring below.

Inventory Monitoring

NTSS is designed to support high volume applications like inventory monitoring. NTSS automatically performs validation checks against all uplinks to check for missed data packets. If a packet was sent from the device but not received by the cloud, NTSS will automatically downlink to the device and request that specific packet again until it has been received.

Static Asset Monitoring

NTSS provides a simple and effective approach to static asset monitoring by leveraging geofencing capabilities to confirm that an asset is still at the expected location. If an asset moves outside of the geofence, NTSS will send an alert that the asset moved. Although more complex and not recommended for indoor use, NTSS can also leverage time difference of arrival (triangulation) to approximate where an asset is within a defined area.

Facility Monitoring

NTSS enables remote configurability and reconfigurability of devices, supporting enterprises with the flexibility they need to support ever changing customer requirements.

Features

Data Loss Protection

Automatic server side acknowledgments and requests for missing data packets.

Remote Configurability

Devices are easily configured and reconfigured by remotely downlinking to the device.

Health Messages

Pre-activation daily health messages provide device health prior to use.

Battery Life Monitoring

Battery voltage is sent with every uplink.

Geofencing

Location confirmation by geofencing around known gateways. Advanced geofencing can be used to approximate distance from gateway.

Battery Drain Mode

Devices can be remotely sent into high power mode to quickly drain remaining battery for safe disposal.
